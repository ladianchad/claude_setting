# 매몰 방지

같은 영역에서 같은 접근을 2회 시도해도 해결 안 되면 (동일 에러 반복, 빌드/테스트 미통과, 가설 검증 실패 등):
1. 즉시 멈춘다. 같은 접근을 3번째 시도하지 않는다.
2. 새 subagent를 생성하여 문제를 위임한다:
   - 전달: 에러 메시지 + 관련 파일 경로 + 이전에 시도한 접근(사실만, e.g. "A 방식으로 수정 시도 → 실패").
   - 전달 금지: 실패 원인에 대한 해석, 자신의 가설, 다음에 시도하려던 방향.
   - subagent는 코드를 처음부터 읽고, 이미 실패한 접근을 피해 독자적으로 원인을 분석한다.
   - subagent는 ~/.claude/rules/principles.md를 읽고 기준으로 삼는다.
3. subagent의 분석이 자신의 접근과 다르면 subagent 방향을 우선 시도한다.
4. subagent도 같은 결론이면 사용자에게 에스컬레이션한다:
   - 시도한 접근 요약 + 실패 이유 + 의심 원인을 보고.
   - 사용자 판단을 기다린다. 독단적으로 계속 시도하지 않는다.
